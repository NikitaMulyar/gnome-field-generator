# gnome-field map-editor

Vue/Vuetify-редактор карты для `gnome-field`.

## Запуск через Docker

Из корня общего проекта `C:\NotGnomes`:

```bash
docker compose up --build
```

Адреса:

```text
game:       http://127.0.0.1:3000/
editor:     http://127.0.0.1:3001/
sync API:   http://127.0.0.1:3002/
```

Если запускается только репозиторий `gnome-field-generator`, команда такая же, но из его корня:

```bash
docker compose up --build
```

## Запуск без Docker

```bash
yarn install
yarn dev
```

Адрес редактора:

```text
http://127.0.0.1:3001/
```

Если нужен base path для GitHub Pages:

```bash
VITE_BASE_PATH=/gnome-field-generator/ yarn dev
```

## Команды

```bash
yarn lint
yarn build
yarn preview
```

`yarn lint` может автоформатировать файлы. После запуска проверяй `git diff`.

## Как пользоваться

- `DimensionControls` задает ширину и высоту поля. По умолчанию поле заблокировано от случайного изменения, кнопку с замком нужно открыть вручную.
- `CellTypesControls` выбирает тип клетки.
- По клеткам можно кликать или зажать левую кнопку мыши и вести курсор по сетке.
- `WallTypeControls` переключает режим стен: верхняя, правая, нижняя, левая или очистка стен.
- В режиме стен клик или протягивание по клеткам ставит выбранную стену.
- Входы и выходы вентиляции создаются 2x2-блоками из клеток `7` и `9`.
- `PortalsControls` показывает найденные 2x2-входы и позволяет выбрать для каждого входа выход.
- `save` скачивает текущую карту в `map.json`.
- `Load JSON` загружает готовую карту из JSON-файла.
- `sync to game` отправляет текущую карту в локальный sync API и обновляет game-проект.
- Кнопка с корзиной очищает все поле, сбрасывает пары порталов и сразу сохраняет пустую карту в автосохранение.

## Загрузка JSON

Кнопка `Load JSON` принимает файл формата `map.json`:

```json
{
  "width": 32,
  "height": 24,
  "portals": [
    {
      "entrance": [1, 2, 33, 34],
      "exit": [100, 101, 132, 133]
    }
  ],
  "tiles": [
    {
      "type": 0,
      "walls": [false, false, false, false]
    }
  ]
}
```

Правила загрузки:

- `width` и `height` должны быть положительными целыми числами.
- `tiles` должен быть массивом длиной `width * height`.
- У каждой клетки используется `type` от `0` до `9`; некорректный тип заменяется на `0`.
- `walls` хранит стены в порядке `up`, `right`, `down`, `left`.
- `portals` необязателен. Если в карте есть соответствующие 2x2-блоки входов и выходов, редактор восстановит связи порталов.
- После успешной загрузки карта сразу становится текущим черновиком и сохраняется в `localStorage`.

Если файл не подходит по формату, редактор покажет ошибку загрузки и не заменит текущую карту.

## Автосохранение

Редактор автоматически сохраняет черновик карты в `localStorage` браузера. Карта переживает:

- обновление страницы;
- закрытие и повторное открытие браузера;
- перезагрузку или выключение компьютера.

Автосохранение срабатывает при изменении размера поля, покраске клеток, изменении стен, загрузке файла, выборе пар порталов и очистке поля.

Черновик привязан к конкретному браузеру и домену. Если открыть редактор в другом браузере, другом профиле или на другом компьютере, там будет отдельное хранилище.

## Рисование протягиванием

Клетки и стены можно ставить не только отдельными кликами. Выбери тип клетки или стены, зажми левую кнопку мыши на поле и веди курсор по сетке. Редактор применит выбранный инструмент ко всем клеткам, через которые проходит курсор.

Автосохранение продолжает работать во время такого рисования.

## Отправка карты в игру

Если проект запущен из корня через общий Docker Compose, в редакторе доступна кнопка `sync to game`.

```bash
docker compose up --build
```

Кнопка отправляет текущую карту в локальный сервис `map-sync-api`:

```text
http://localhost:3002/sync-map
```

Sync API сохраняет `map.json`, запускает генерацию `map.png`, а потом копирует оба файла в игру:

```text
src/assets/map.json -> ../gnome-field/gnome-field/public/map.json
out/map.png         -> ../gnome-field/gnome-field/src/assets/map.png
```

После успешной синхронизации обнови страницу игры на `http://localhost:3000/`.

Если редактор запущен отдельно без общего compose и без sync API, кнопка покажет ошибку подключения.

URL sync API можно переопределить через переменную окружения:

```bash
VITE_MAP_SYNC_URL=http://localhost:3002/sync-map yarn dev
```

## Типы клеток

| Код | Значение |
| --- | --- |
| `0` | вода |
| `1` | листочки |
| `2` | дверь в подвал |
| `3` | булочка |
| `4` | банка краски |
| `5` | картон |
| `6` | сканер |
| `7` | вход в вентиляцию |
| `8` | волшебная коробка |
| `9` | выход из вентиляции |

## Стены

Стены хранятся в каждой клетке массивом:

```json
"walls": [false, false, false, false]
```

Порядок направлений:

```text
0 = up
1 = right
2 = down
3 = left
```

Если поставить стену справа у одной клетки, редактор не ставит автоматически зеркальную стену слева у соседней. Для игровой логики лучше ставить обе стороны вручную, если нужна полностью закрытая граница.

## Порталы

Редактор ищет порталы не по одиночным клеткам, а по квадратам 2x2:

```text
7 7      9 9
7 7      9 9
```

`7` - вход, `9` - выход. После того как такие блоки нарисованы, блок `Portals` позволяет связать каждый вход с выходом. При сохранении связь попадет в поле `portals`.
